## 🎯 关键设计决策

### 1. 为什么选择全宽Hero + 渐变背景？

**优势**：
- ✅ 第一印象强烈，视觉冲击力大
- ✅ 突出个人品牌
- ✅ 比纯文字更现代
- ✅ 保持学术专业感（非过度设计）

**对比**：
- Jason Wei：简洁但视觉平淡
- Yingchao Li：需要高质量背景图片
- 本设计：渐变背景易维护，视觉效果好

### 2. 为什么使用卡片式论文展示？

**优势**：
- ✅ 视觉层次清晰
- ✅ 会议徽章醒目（NeurIPS、AAAI等）
- ✅ 悬停动画增强交互感
- ✅ 便于展示多个链接（PDF、Code、Media）

**对比**：
- 表格式：信息密度高但视觉吸引力弱
- 卡片式：平衡了美观和信息展示

### 3. 为什么添加深色模式？

**优势**：
- ✅ 现代网站标配
- ✅ 夜间阅读友好
- ✅ 符合开发者/研究者习惯
- ✅ 展示技术能力

### 4. 为什么不使用公司Logo？

**理由**：
- 你目前主要是学生+实习生身份
- 论文成果更重要
- 未来可以在Experience section添加Logo

## 🎭 视觉特效

### 交互动画

1. **卡片悬停效果**
   - 上移 3-5px
   - 阴影加深
   - 边框变色（主色调）

2. **滚动渐入动画**
   - Intersection Observer API
   - 从透明+下移 → 完全显示
   - 自然流畅

3. **导航栏滚动效果**
   - 滚动>100px时添加阴影
   - 增强层次感

4. **Hero区域视差**
   - 轻微的向下移动
   - 增加深度感

## 📱 响应式设计

### 断点设计

```css
/* 桌面（默认）：>1024px */
- 完整布局
- 卡片网格：2-3列
- 导航栏完整显示

/* 平板：768px - 1024px */
- 卡片网格：2列
- 字体略小
- 间距调整

/* 手机：<768px */
- 单列布局
- Hero标题：2.5rem
- 导航栏简化（未来可添加汉堡菜单）
```

## 🚀 性能优化

1. **字体加载**：使用系统字体栈，无需外部加载
2. **CSS变量**：便于主题切换，减少代码重复
3. **Font Awesome CDN**：仅加载需要的图标
4. **渐进增强**：基础内容优先，动画为增强

## 🎓 设计哲学

> **"简约而不简单，专业而有个性"**

- **简约**：没有多余装饰，突出内容
- **不简单**：精致的动画、完善的交互
- **专业**：学术风格、清晰的信息架构
- **个性**：渐变设计、主题切换、会议徽章
